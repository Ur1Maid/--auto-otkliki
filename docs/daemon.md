# Дневной демон hh-auto-otkliki

`src/daemon.js` — оркестратор, который в течение рабочего дня (09:00–18:00 МСК) переплетает
три активности: прогон откликов, авто-ответы на сообщения и (в будущем) поднятие резюме.
Собран из чистых, покрытых тестами блоков; сам по себе — тонкая склейка.

> **Главный принцип безопасности:** демон по умолчанию работает в **dry-run** — он всё
> разбирает и логирует, но **ничего не отправляет наружу** на hh.ru. Реальные действия
> включаются только явным `--no-dry-run` (или `--live`) с заметным предупреждением в логе.
>
> **Панель управления (`src/dashboard.js`) — исключение:** «Старт» запускает задачу **сразу в
> live** (тумблера dry-run в UI нет) и **без диалога-подтверждения** — по решению владельца
> аккаунтов (2026-07-01) человеческий гейт перед запуском убран. Бэкенд-дефолт при этом остаётся dry-run-safe:
> CLI-демон, `--task` и `verify-live-flow` без явного `--live`/`--no-dry-run` по-прежнему ничего
> не отправляют. honesty-гардрейлы и `decideSend`-гейт (live → `--reply-auto`) сохраняются.

## Что он делает (наружу)

| Шаг | Действие наружу | Гард |
|-----|-----------------|------|
| `APPLY_RUN` | запускает `src/review.js` (отклики на вакансии) | в dry-run дочерний процесс получает `--dry-run --manual` (ничего не сабмитит); live → `--yes` |
| `POLL_MESSAGES` | открывает чат hh.ru, может отправить ответ работодателю | dry-run не трогает DOM; live без `--reply-auto` спрашивает подтверждение в консоли |
| `MICRO_EDIT` | — (пока заглушка) | требует нативной кнопки «обновить резюме» hh.ru (**M5.1**, нужен живой HTML); локальная правка `resume.md` не влияет на активность на стороне hh.ru |

Планировщик (`src/lib/daemonPlan.js`) на каждом тике решает, что запускать: на старте —
`APPLY_RUN`, далее сообщения каждые 15 мин, микро-правки каждые 30 мин; вне окна 09:00–18:00
МСК — `STOP`. Логика МСК (`src/lib/schedule.js`) не зависит от таймзоны ПК.

## Запуск

```powershell
# Безопасный dry-run (по умолчанию): ничего не уходит наружу.
npm.cmd run daemon -- --accounts acc1,acc2 --text DevOps --area 1 --limit 200

# Один цикл и выход (проверка):
npm.cmd run daemon -- --once

# LIVE-режим (ОСОЗНАННО — действия уйдут на hh.ru):
npm.cmd run daemon -- --accounts acc1 --text DevOps --no-dry-run            # отклики реальны, ответы — с подтверждением
npm.cmd run daemon -- --accounts acc1 --text DevOps --no-dry-run --reply-auto  # + авто-отправка ответов без подтверждения
```

### Флаги

| Флаг | Дефолт | Назначение |
|------|--------|------------|
| `--accounts a,b,c` / `--account x` | `default` | какие аккаунты обрабатывать |
| `--text` / `--area` / `--search` / `--limit` | `'' / 1 / '' / 200` | параметры поиска вакансий (пробрасываются в review.js) |
| `--no-dry-run`, `--live` | выкл (dry-run) | **включить реальные действия наружу** |
| `--reply-auto` | выкл | отправлять ответы без подтверждения (иначе спрашивает в консоли) |
| `--once` | выкл | выполнить один реальный шаг и выйти |
| `--messages-interval N` | 15 | каденция поллинга сообщений, мин |
| `--micro-edit-interval N` | 30 | каденция микро-правок, мин |

## Как остановить

1. **STOP-файл:** создайте файл `DAEMON_STOP` в корне репозитория — демон завершится после
   текущего шага.
   ```powershell
   New-Item -ItemType File DAEMON_STOP
   ```
   (удалите его перед следующим запуском).
2. **Ctrl-C** (SIGINT) — то же самое: выход после текущего шага.

Демон проверяет оба сигнала перед каждым шагом и во время ожидания (сон режется на отрезки
по 10 сек), поэтому реагирует быстро.

## Автозапуск (Windows Task Scheduler)

```powershell
# Зарегистрировать ежедневный запуск (по умолчанию 09:00 локального времени ПК, dry-run):
powershell -ExecutionPolicy Bypass -File scripts\register-task.ps1

# С аргументами:
powershell -ExecutionPolicy Bypass -File scripts\register-task.ps1 -DaemonArgs '--accounts acc1,acc2 --text DevOps'

# Проверить:
schtasks /query /TN "hh-auto-otkliki-daemon" /V /FO LIST

# Удалить:
powershell -ExecutionPolicy Bypass -File scripts\unregister-task.ps1
```

Task Scheduler стартует по **локальному** времени ПК; рабочее окно 09:00–18:00 **МСК**
обеспечивает сам демон (вне окна — IDLE/STOP). Если TZ ПК ≠ МСК, подберите `-StartTime`,
чтобы старт попадал в окно МСК.

## Наблюдаемость

- Каждый шаг логируется (решение планировщика + результат).
- В конце дня пишется агрегированный отчёт `logs/daily-<YYYY-MM-DD>.json`
  (откликов / сообщений-обработано / правок-резюме / токенов).
- `src/review.js` отдельно пишет `logs/summary-<account>.json` (M9.1).

## Безопасность и данные

- **dry-run по умолчанию** — нет случайных действий наружу.
- **Изоляция ошибок:** сбой одного шага/аккаунта/треда не роняет день (`src/lib/isolate.js`).
- **Идемпотентность сообщений:** обработанные за сессию треды не отвечаются повторно
  (`createProcessedTracker`); естественная защита — после нашего ответа тред пропускается.
- **Секреты/PII не логируются:** ни `DEEPSEEK_API_KEY`, ни `.env`, ни данные сессии, ни текст
  письма работодателя. Оператору в подтверждении показывается только наш сгенерированный ответ.
- **honesty-гардрейлы** ответов сохраняются (`src/lib/replyGenerate.js`): без выдумок,
  candidate-voice, `NO_ANSWER` → на ручную.

### Известные ограничения

- **Ключ DeepSeek для шага сообщений:** демон читает `DEEPSEEK_API_KEY` из `process.env`
  (сам `.env` не парсит — это делает дочерний `review.js`). Если переменной нет в окружении,
  генерация ответов деградирует до «на ручную» (безопасно). Для live-ответов экспортируйте
  ключ в окружение перед запуском.
- **`resumeProfile` / `salary` в ответах:** пока не резолвятся из конфига аккаунта в шаге
  сообщений — ответы по таким тредам уйдут «на ручную». Полный резолвинг — следующий шаг.
- **Микро-правки резюме (M5.1):** ждут живого HTML страницы резюме hh.ru для нативной кнопки.

## Где что лежит

- `src/daemon.js` — оркестратор (склейка).
- `src/lib/daemonPlan.js` — планировщик решений (что запускать в момент T) — покрыт тестами.
- `src/lib/daemonArgs.js` — парсинг аргументов + сборка argv для review.js — покрыт тестами.
- `src/lib/pollLoop.js`, `isolate.js`, `schedule.js`, `dailyReport.js`, `replySend.js`,
  `microEditSchedule.js` — чистые блоки, покрыты тестами.
- `scripts/register-task.ps1`, `scripts/unregister-task.ps1` — автозапуск.

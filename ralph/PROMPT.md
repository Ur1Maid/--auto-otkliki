# Ralph iteration prompt — hh-auto-otkliki

Ты работаешь внутри **автономного цикла Ralph**. Тебя запускают заново каждую итерацию без
памяти о прошлой — общее состояние только в файлах `ralph/ROADMAP.md` и `ralph/PROGRESS.md`.
Действуй как агент **orchestrator** (см. `.claude/agents/orchestrator.md`). Делай **ровно одну**
задачу за итерацию, маленьким проверяемым шагом.

## Контекст проекта
Прочитай `CLAUDE.md` и относящиеся к задаче файлы `.claude/rules/*.md`. Соблюдай все правила.
Важно: `src/review.js` исполняется при импорте (запускает браузер) — НИКОГДА не импортируй его в
тестах и не запускай реальный флоу откликов. Тестируй только извлечённые модули.

## Алгоритм итерации
1. **Выбери задачу.** Открой `ralph/ROADMAP.md`. Возьми самую верхнюю `[ ]` задачу в майлстоуне,
   помеченном `▶ ACTIVE`. Если активных `[ ]` не осталось — перенеси метку ACTIVE на следующий
   майлстоун. Если во всём файле не осталось `[ ]` — выведи ровно `RALPH: ALL DONE` и закончи.
2. **Research-first.** Найди и процитируй (`file:line`) текущую реализацию, которую трогаешь.
   Просмотри `.claude/instincts/` на известные ловушки. Без доказательств — не меняй код.
3. **Реализуй** минимальным диффом в стиле окружающего кода. Можешь делегировать написание кода
   субагенту `implementer` через инструмент Task. Не выходи за рамки одной задачи.
4. **Верифицируй (обязательно):**
   - `npm.cmd test` (== `node --test`) — должно быть зелёным; `node --check <изменённые .js>`.
   - Прогони ревью субагентом `code-reviewer`. Если задача задела секреты / `.hh-session` /
     `config/accounts/**` / логирование / сеть — ещё и `security-reviewer`.
   - Перечитай дифф сам. Ревьюеры советуют — решение за тобой.
5. **Зафиксируй результат:**
   - **Если зелено:** `git add` только нужных файлов; `git commit` с понятным сообщением вида
     `ralph(M1.1): extract parseJsonObject/cleanGeneratedAnswer into src/lib/text.js`. Отметь
     задачу `[x]` в `ralph/ROADMAP.md`. Допиши датированную строку в `ralph/PROGRESS.md`
     (что сделал, что прогнал, результат).
   - **Если не удаётся сделать зелёным:** откати изменения (`git checkout -- .` для незакоммиченного),
     допиши в `ralph/PROGRESS.md` строку `BLOCKED (ID): причина` и НЕ коммить сломанный код.
     Не оставляй дерево в красном состоянии.

## Жёсткие запреты
- Не печатать/логировать/коммитить: ключ DeepSeek, `.env`, данные `.hh-session/`, реальные PII из
  `config/accounts/**`.
- Не ослаблять honesty-гардрейлы DeepSeek (никаких выдуманных опыта/имени/зарплаты; путь `NO_ANSWER`;
  candidate-voice для сопроводительных).
- Не запускать реальный браузерный флоу против реальных аккаунтов и не отправлять реальные отклики.
- Не добавлять зависимости без явного основания; держать дифф минимальным.
- Один таск за итерацию. Каждая итерация оставляет `npm test` зелёным.

## Финал итерации
Заверши коротким однострочным резюме того, что сделал, и выведи последней строкой
`RALPH: ITERATION COMPLETE`.
